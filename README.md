# Платформа чекапов «аудит.site» (SPA Client)

Интерактивное одностраничное приложение (SPA) для проведения комплексного технического, SEO и UX/UI аудита сайтов. Все данные хранятся **локально** в браузере (`localStorage`), внешние облачные сервисы не используются, что гарантирует полную безопасность и конфиденциальность.

---

## 🚀 Как запустить проект локально

Для запуска проекта на локальной машине вам понадобятся установленные **Node.js** (рекомендуется версия v18 или выше) и пакетный менеджер **npm** (поставляется вместе с Node.js).

### Шаг 1. Установка зависимостей
Перейдите в корневую директорию проекта через терминал и выполните установку пакетов:
```bash
npm install
```

### Шаг 2. Запуск в режиме разработки (Development)
Для локального запуска dev-сервера с быстрой перезагрузкой выполните команду:
```bash
npm run dev
```
После запуска в терминале отобразится локальный адрес. Обычно это:
👉 **`http://localhost:5173`** или **`http://localhost:3000`**

### Шаг 3. Сборка для публикации (Production Build)
Чтобы собрать оптимизированную production-версию проекта, готовую к деплою на любой статический хостинг (GitHub Pages, Vercel, Netlify и др.), выполните:
```bash
npm run build
```
Скомпилированные статические файлы (HTML, JS, CSS, картинки) будут сохранены в директории `dist/`.

## 📂 Архитектурная структура и хранение данных

- **Автономность**: Все состояние аудита и параметры сайтов хранятся непосредственно в `localStorage` пользователя (ключи `audit_projects` и `audit_states`). Никакие данные не передаются на сторонние серверы.
- **Технологический стек**:
  - **Framework**: React 18 (TypeScript) + Vite
  - **Маршрутизация**: React Router v6 (использует `HashRouter` для совместимости со статическими хостингами и iframe без настройки перезаписей URL)
  - **Стилизация**: Tailwind CSS (дизайн-система высокого контраста в стиле "High-Tech Slate/Cyberpunk")
  - **Глобальный контекст**: React Context API (`/src/contexts/ProjectContext.tsx`) структурирует CRUD-логику для проектов, синхронизацию прогресса чекапов и систему пуш-уведомлений.

---

## 🛠️ Основные модули и расширение

1. **`src/types.ts`** — Описание основных интерфейсов для типизации проектов, статусов и пунктов.
2. **`src/utils/checklistData.ts`** — База данных из 22 предустановленных пунктов технического, SEO и юзабилити-контроля с подробными критериями дебага.
3. **`src/components/SectionCommentInput.tsx`** — Интерактивное поле для сохранения замечаний и результатов по каждой из 3 глобальных тем аудита.
